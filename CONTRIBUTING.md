# Contributing

Thanks for your interest in contributing.

## Getting started
- Fork the repository and create a feature branch.
- Install dependencies with `npm install`.
- Run local checks with `npm run ci:local`.

## Pull requests
- Keep PRs focused and linked to an issue when possible.
- Include test coverage for behavior changes.
- Update docs for user-visible or operational changes.

## Commit and coding standards
- Use clear commit messages describing intent.
- Keep TypeScript strict and avoid `any` where possible.
- Follow lint and typecheck requirements before opening PRs.

## Security and sensitive data
- Never commit secrets, API keys, or private user data.
- Follow `SECURITY.md` for vulnerability disclosure.
